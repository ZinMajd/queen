# وثيقة تصميم وعمل النظام - منصة خدمات الزفاف (Queen)

## 🟣 أولاً: وصف عمل النظام (System Workflow)
النظام عبارة عن منصة إلكترونية لإدارة خدمات الزفاف، ويعمل كالتالي:

### 🔹 خطوات العمل العامة:
1. دخول المستخدم (زائر أو تسجيل)
2. تحديد نوع المستخدم:
   - عروس
   - مزود خدمة
   - إدارة
3. تنفيذ العمليات حسب الدور

---

### 👰‍♀️ تدفق عمل العروس:
1. اختيار المناسبة
2. تصفح الفساتين والخدمات
3. عرض التفاصيل
4. الحجز
5. اختيار التوصيل
6. الدفع
7. متابعة الطلب

---

### 💄 تدفق مزود الخدمة:
1. تسجيل حساب
2. انتظار الموافقة
3. إضافة خدمات
4. استقبال الحجوزات
5. الرد على العملاء

---

### 🛠️ تدفق الإدارة:
1. إدارة المستخدمين
2. قبول/رفض مزودي الخدمات
3. إدارة الفساتين
4. متابعة الحجوزات
5. مراقبة النظام

---

## 🟢 ثانياً: واجهات النظام (UI) + العمليات داخل كل واجهة

### 🏠 1. الصفحة الرئيسية
- **المكونات:** صورة رئيسية، تعريف بالمحل، اختيار المناسبة، أزرار تسجيل / دخول.
- **العمليات:** اختيار (خطوبة / عقد / عرس)، الانتقال للتصفح، تسجيل / دخول.

### 🔐 2. تسجيل الدخول
- **العمليات:** إدخال البيانات، التحقق (Validation)، تسجيل الدخول، تحويل حسب الدور.

### 📝 3. إنشاء حساب
- **العمليات:** إدخال البيانات، تحديد نوع المستخدم، حفظ البيانات، إذا مزود خدمة → حالة "قيد المراجعة".

### 👗 4. صفحة الفساتين
- **العمليات:** عرض الفساتين، فلترة (نوع، مقاس، حالة)، البحث، فتح التفاصيل.

### 📄 5. صفحة تفاصيل الفستان
- **العمليات:** عرض التفاصيل، عرض الصور، تواصل (واتساب / اتصال)، بدء الحجز.

### 📅 6. صفحة الحجز
- **العمليات:** اختيار التاريخ، إضافة خدمات إضافية (تاج، ورد، توصيل)، تأكيد الحجز.

### 💄 7. صفحة الخدمات
- **العمليات:** عرض الخدمات، اختيار مزود، حجز الخدمة.

### 🚚 8. صفحة التوصيل
- **العمليات:** اختيار (استلام، توصيل)، إدخال العنوان.

### 💳 9. صفحة الدفع
- **العمليات:** اختيار طريقة الدفع، تأكيد الدفع، تحديث الحالة.

### 📊 10. صفحة حجوزاتي
- **العمليات:** عرض الحجوزات، متابعة الحالة، إلغاء الحجز.

### 💼 11. لوحة مزود الخدمة
- **العمليات:** 
  - **إدارة الخدمات:** إضافة، تعديل، حذف.
  - **إدارة الحجوزات:** عرض، قبول، رفض.

### 🛠️ 12. لوحة الإدارة
- **العمليات:** 
  - **إدارة المستخدمين:** عرض، حذف.
  - **مزودي الخدمات:** قبول / رفض.
  - **الفساتين:** إضافة / تعديل / حذف.
  - **الحجوزات:** متابعة / إلغاء.

---

## 🔵 ثالثاً: منطق النظام (Business Logic)

### قواعد مهمة جداً:

**الحجز:**
- لا يمكن حجز فستان إذا كان محجوزاً بنفس التاريخ ❌
- يجب اختيار تاريخ صحيح ✅

**مزود الخدمة:**
- لا يمكنه الدخول إلا بعد الموافقة ❌
- لا يمكن إضافة خدمات بدون تفعيل الحساب ❌

**الدفع:**
- لا يتم تأكيد الحجز إلا بعد الدفع (أو اختيار الدفع عند الاستلام).

**التوصيل:**
- إذا اختار توصيل → يجب إدخال عنوان.

---

## 🔐 رابعاً: نظام الأمان (Security)
1. **المصادقة (Authentication):** تسجيل دخول، تسجيل خروج، جلسات.
2. **الصلاحيات (Authorization):**
   - العروس: حجز فقط.
   - المزود: إدارة خدماته فقط.
   - الإدارة: تحكم كامل.
3. **حماية البيانات:** تشفير كلمات المرور، حماية API، منع SQL Injection.

---

## 🔔 خامساً: نظام الإشعارات
- **إشعارات:** تأكيد الحجز، رفض الحجز، قبول مزود خدمة.

---

## 📅 سادساً: نظام إدارة المواعيد
- **العمليات:** اختيار تاريخ، عرض الأيام المتاحة، منع الحجز المكرر.

---

## ⭐ سابعاً: نظام التقييم
- **العمليات:** تقييم مزود الخدمة، تقييم الفستان، عرض التقييم.

---

## ❤️ ثامناً: المفضلة
- **العمليات:** إضافة للمفضلة، حذف، عرض القائمة.

---

## 📊 تاسعاً: التقارير (للإدارة)
- **العمليات:** عرض (عدد الحجوزات، الخدمات الأكثر طلباً، نشاط المستخدمين).

---

## ⚙️ عاشراً: إعدادات النظام
- **العمليات:** تعديل بيانات الموقع، تحديث رقم التواصل، إدارة الأقسام.

---

## 🔍 حادي عشر: البحث المتقدم
- **العمليات:** البحث بالاسم، فلترة (نوع، مقاس، تقييم، حالة).

---

## ⚙️ ثاني عشر: العمليات التقنية (Backend)

- **API أمثلة:**
  - `POST /login`
  - `POST /register`
  - `GET /dresses`
  - `POST /booking`

- **التحقق (Validation):** منع الحقول الفارغة، التحقق من رقم الهاتف، التحقق من التاريخ.
- **رفع الصور:** فساتين، خدمات، حسابات.
- **تقسيم عرض البيانات (Pagination):** تقسيم عرض البيانات عند الكثرة.
- **تجربة المستخدم (UX):** رسائل نجاح، رسائل خطأ، شاشة تحميل.
