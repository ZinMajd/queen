# ملخص مشروع منصة الملكة (Queen Platform)

## 1. التقنيات واللغات المستخدمة
**الواجهة الخلفية (Backend):**
- **إطار العمل:** Laravel (إصدار 12)
- **لغة البرمجة:** PHP (إصدار 8.2 وما فوق)
- **المصادقة:** Laravel Sanctum
- **قاعدة البيانات:** MySQL / SQLite (حسب الإعداد في ملف .env)

**الواجهة الأمامية (Frontend):**
- **إطار العمل:** React (مبني باستخدام Vite)
- **لغة البرمجة:** JavaScript
- **التصميم وتنسيق الواجهات:** Tailwind CSS
- **إدارة المسارات (Routing):** React Router DOM
- **التواصل مع الخادم:** Axios

---

## 2. عدد جداول قاعدة البيانات وما هي
يحتوي النظام على **11 جدولاً** في قاعدة البيانات بناءً على ملفات التهيئة (Migrations)، وهي:
1. `users`: لتخزين بيانات المستخدمين.
2. `vendors`: لتخزين بيانات مزودي الخدمات.
3. `categories`: لتصنيفات الفساتين والخدمات.
4. `dresses`: لتخزين بيانات الفساتين المتاحة بالمنصة.
5. `services`: لتخزين بيانات الخدمات الإضافية المتاحة.
6. `bookings`: لتسجيل وإدارة الحجوزات التي يطلبها المستخدمون.
7. `deliveries`: لإدارة عمليات التوصيل المرتبطة بالحجوزات.
8. `payments`: لتسجيل وتتبع عمليات الدفع.
9. `personal_access_tokens`: الرموز المميزة (Tokens) لإدارة جلسات المصادقة (Sanctum).
10. `cache`: جدول افتراضي لحفظ التخزين المؤقت.
11. `jobs`: جدول افتراضي لإدارة المهام المنفذة في الخلفية.

---

## 3. العمليات الموجودة في المشروع وكيف يعمل
يقوم المشروع بتشغيل **منصة لحجز فساتين الزفاف والخدمات المرافقة لها**. يعمل النظام عن طريق واجهة أمامية (React) تتخاطب مع واجهة برمجة تطبيقات (Laravel API).

**أهم العمليات (الـ API Routes) المتاحة في النظام:**
- **المصادقة وإدارة الحسابات:**
  - تسجيل حساب جديد للعروس أو المستخدم (`POST /register`)
  - تسجيل الدخول (`POST /login`)
  - عرض بيانات المستخدم المسجل (`GET /user`)
  - تسجيل الخروج (`POST /logout`)
  
- **إدارة الحجوزات (Bookings):**
  - استعراض جميع حجوزات المستخدم (`GET /bookings`)
  - تقديم طلب حجز جديد لفستان أو خدمة (`POST /bookings`)
  - عرض تفاصيل حجز معين وحالته (`GET /bookings/{id}`)
  
- **الاستعراض والتصفح (للعملاء):**
  - استعراض التصنيفات المختلفة (`GET /categories`) وتفاصيل تصنيف محدد.
  - استعراض جميع الفساتين المتاحة (`GET /dresses`) وعرض تفاصيل كل فستان للتمكن من اختياره وحجزه.
  - تصفح الخدمات الإضافية المتاحة (`GET /services`).

**آلية العمل باختصار:**
يدخل المستخدم (العروس) إلى الموقع، يتصفح الفساتين والتصنيفات والخدمات بدون الحاجة لتسجيل دخول مبدئياً. عند رغبته في الحجز، يقوم بالنقر على إتمام الحجز، مما يتطلب منه تسجيل الدخول أو إنشاء حساب (عبر Sanctum). بعد الدخول، يقوم النظام بإرسال طلب الحجز للواجهة الخلفية (Laravel) التي تحفظه في جدول `bookings` وتمرره لمراحل الدفع (`payments`) والتوصيل (`deliveries`) بالاعتماد على مزودي الخدمة (`vendors`).
