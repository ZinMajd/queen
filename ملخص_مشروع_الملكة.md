# ملخص شامل لمشروع "منصة الملكة" (Queen Platform)

تم إعداد هذا الملخص للمشروع خلال دقيقة، وهو يوضح هيكلية النظام وآلية عمله.

---

## 1. ما هي العمليات الموجودة في المشروع وكيف يعمل؟
المشروع عبارة عن **منصة إلكترونية متكاملة لحجز فساتين الزفاف وطلب الخدمات المرافقة** الخاصة بالعرائس. يعمل النظام من خلال تفاعل بين واجهة المستخدم الأمامية (تصفح، اختيار، حجز) والواجهة الخلفية (تأكيد، دفع، إدارة).

**كيف يعمل (دورة حياة الطلب):**
1. **التصفح:** يقوم المستخدم (أو العروس) بتصفح واجهة الموقع لرؤية الفساتين المتاحة (`/dresses`) والخدمات الأخرى (`/services`) حسب التصنيفات (`/categories`).
2. **التسجيل / الدخول:** في حال الرغبة بالحجز، يقوم المستخدم بتسجيل حساب جديد أو الدخول لحسابه (تتم المصادقة عبر Sanctum).
3. **الحجز:** عند تأكيد الحجز، يرسل النظام طلبا (POST `/bookings`) لقاعدة البيانات.
4. **المعالجة:** يتم ربط الحجز بمقدم الخدمة (`vendor`)، وتسجيل معلومات الموقع للتوصيل (`delivery`)، وإتمام وتحقق من الدفع (`payment`).

---

## 2. التقنيات واللغات المستخدمة
**في قسم الخادم والواجهة الخلفية (Backend):**
- **التقنية / إطار العمل:** Laravel 12 (من أقوى أطر العمل لـ PHP).
- **اللغة المستخدمة:** PHP 8.2 (أو أحدث).
- **تقنية أمان الـ API:** Laravel Sanctum للتحقق من هوية المستخدمين والمصادقة.

**في قسم واجهة المستخدم (Frontend):**
- **التقنية / إطار العمل:** مكتبة React.js للواجهات التفاعلية، مدعومة بأداة Vite لتسريع بناء وتطوير المشروع.
- **اللغة المستخدمة:** JavaScript (مع JSX).
- **تصميم الواجهات:** Tailwind CSS (لكتابة التنسيقات بشكل سريع واحترافي).
- **إدارة مسارات الصفحات:** React Router (للتنقل بين الصفحات في المتصفح بسلاسة).

---

## 3. كم عدد جداول قاعدة البيانات وما هي؟
يحتوي النظام حالياً على **11 جدولاً** جاهزاً في قاعدة البيانات، تنقسم إلى جداول أساسية لعمليات النظام وجداول تابعة لإطار Laravel:

**جداول النظام الأساسية (8 جداول):**
1. **`users`**: لتسجيل بيانات جميع المستخدمين والعملاء النشطين (الاسم، الإيميل، الباسورد، إلخ).
2. **`vendors`**: لتسجيل مزودي الخدمات (مثل أصحاب محلات الفساتين أو مقدمي الخدمات).
3. **`categories`**: تصنيف الفساتين والخدمات بناءً على الفئة.
4. **`dresses`**: بيانات شاملة لجميع الفساتين المعروضة للإيجار أو البيع.
5. **`services`**: بيانات الخدمات الأخرى المتاحة للعرائس.
6. **`bookings`**: الجدول المحوري الذي يسجل تفاصيل كل عملية حجز تتم في المنصة وربطها بالمستخدم ومقدم الخدمة.
7. **`deliveries`**: لتتبع حالات تسليم الطلب ومواقع التوصيل.
8. **`payments`**: لتسجيل بيانات عمليات الدفع (الدفع الإلكتروني أو غيره) المرتبطة بالحجوزات.

**جداول عمليات إطار العمل والتحقق (3 جداول):**
9. **`personal_access_tokens`**: توليد والاحتفاظ برموز الدخول (Tokens) الخاص بالمصادقة وجلسات المستخدمين.
10. **`cache`**: لحفظ وتخزين البيانات المؤقتة لتسريع الأداء.
11. **`jobs`**: لجدولة وإدارة مهام النظام الخلفية مثل إرسال الإيميلات وغيرها.
